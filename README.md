#  Тамагочи: Лунис и Kai Angel

Учебный проект по созданию виртуального питомца с использованием JavaScript (замыкания и работа с DOM).

## 👤 Персонажи (Кто они)

В игре доступно два персонажа на выбор:

1. **Лунис (Основной персонаж)** 
   Неко-девочка с кошачьими ушками и хвостом. Она игривая и любопытная, но требует внимательной заботы. Быстро устаёт, постоянно забывает пить и очень привередлива в еде.
2. **Kai Angel (Сложный уровень)** 🦇
   Альтернативный персонаж для тех, кто ищет вызов. Характеризуется экстремально высокой скоростью сжигания калорий и частой жаждой. Требует от игрока гораздо больше внимания и невероятно быстрого кликанья по кнопкам.

---

##  Правила убывания статов и действий

-  **Старт:** Начальное значение всех характеристик (Вода, Еда, Энергия) равно **100**.
-  **Тик (Убывание):** Все характеристики убывают каждую секунду на определенное значение, которое зависит от выбранного персонажа.
-  **Действия:** Нажатие на кнопки действий («Пить», «Есть», «Отдохнуть») пополняет соответствующий стат на заданное количество единиц.
-  **Лимиты:** Значение характеристик всегда ограничено диапазоном от **0 до 100** включительно (реализована функция `clamp`).
-  **Смерть:** Если хотя бы один из статов падает до **0**, персонаж погибает:
  - Все кнопки действий блокируются.
  - Таймер жизненного цикла останавливается.
  - Появляется окно с сообщением о смерти персонажа и предложением начать игру заново.

##  Выполненные усложнения (на оценку "5")

В проекте реализованы все дополнительные механики для получения максимальной оценки:

### 1.  Конфигурация персонажа через объект
Фабрика `createCharacter(config)` переписана таким образом, чтобы принимать объект с настройками. Это позволило создать разных персонажей с уникальным балансом.
**Конфиг задает:**
- `name` — имя персонажа.
- `stats` — стартовые значения (вода, еда, энергия).
- `decay` — значение убывания статов за один `tick`.
- `actions` — сила действий (на сколько единиц увеличиваются статы при клике).

### 2.  Зависимость между статами (Условные эффекты)
Реализовано влияние одного стата на другой.
- Если показатель **воды** (`water`) опускается **ниже 20**, то **энергия** (`energy`) начинает убывать **в 2 раза быстрее** (штраф к скорости потери энергии в методе `tick`).

### 3.  Защита логики после гибели персонажа
Реализована строгая блокировка состояния "Мертв", использующая механику замыканий.
- В методы `drink()`, `eat()`, `rest()` и `tick()` внедрена проверка: `if (water === 0 || food === 0 || energy === 0) return;`.
- Благодаря этому, после смерти значения статов полностью "заморожены". Их невозможно изменить ни кликами, ни программно из консоли браузера, пока страница не будет перезагружена.
